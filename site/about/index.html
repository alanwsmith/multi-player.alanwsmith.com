About Multi-Player

Overview

This project is a recreation of a page I saw years ago. It used the Alice video from Pogo. I can't find the original. So, I decided to make a new version.

The Audio

I initially had the audio coming from the first frame in the upper left. It works for some videos. Mostly, I found it nicer to move the audio to sync with a more central frame. It's currently set to come from the middle column on one of the first three rows (depending on the total number of rows).

That's where I found my eyes drifting most of the time.

I played around with highlighting the audio source frame in different ways. All of them felt more distracting from the overall experience than they were worth.

Tech Details

YouTube API

The videos come from YouTube via their iFrame API. It lets you embed videos on a page and control them with JavaScript.

While the iFrame API is pretty solid, it's missing a couple 'nice-to-haves'. Specifically, the title and duration of the video. Displaying the title while the video is loading would be a nice touch. With the duration, I could add a nicer fade out at the end.

Those details are available in other YouTube APIs. Adding them was more complexity than I wanted to get into at the moment.

Web Components

The page is built from vanilla web components. Using components is maybe overkill. I'm still new to them though. This was a nice way to get some reps under the belt on a small project.

Syncing The Videos

The page automatically determines how may players to use based on the browser's window size. The videos start with a little delay between each one. It works well most of the time on good network connections. Sometimes they don't start when they should and things look a wonky. If you're on a solid network connection refreshing the browser might make things smoother.

I've got some ideas about how to ping the video players and re-sync them while they are playing. I'm not sure if it'll be possible with the timing from the API though.

There may be an approach to this that uses a single video and somehow renders it multiple times. Something else to maybe investigate in the future (if you know how to do that, please let me know).

Two Patterns

If you look at the source code you'll see commented out sections for another display pattern. It's more of star where things expand out from a central frame. I liked the idea, but it didn't look as good when I implemented it.

I went back to the filmstrip style. I might play around with other patterns at some point. I've got some ideas for that.

Future Enhancements

I've got a few ideas for features to add. But, I'm happy with where it is now. I want to roll off to work on other projects. I'll probably come back to this at some point. For now, I hope you enjoy it!
